## 02:20 修复 SDK 群事件监听重复出卡

- 修改内容：
  - 调整 `src/chatEventMain.js` 的参数解析：当 `--event-source sdk` 时，如果用户没有显式传入 `--reconcile-poll` / `--no-reconcile-poll`，默认关闭历史消息轮询补偿。
  - 保留 `lark-cli` 事件源下默认开启 reconcile 的旧行为。
  - 更新 `README.md` 中群事件参数说明，明确 `sdk` 模式默认不启用 reconcile，`lark-cli` 模式默认启用。
- 修改原因：
  - 官方 SDK WebSocket 已经成为主监听链路，继续默认启用轮询补偿会导致同一条群消息可能被 SDK 和轮询各处理一次。
  - 用户调试时常加 `--push-bypass-dedupe`，会绕过推送层去重，使重复处理直接表现为两张知识卡片。
- 影响范围：
  - 只影响 `src/chatEventMain.js` 的 SDK 事件监听默认行为。
  - 如果确实需要在 SDK 模式下继续做轮询补偿，可以显式加 `--reconcile-poll`。
- 验证：
  - 已进行语法检查。
