## 23:27

### 修改内容描述

- 新增 `src/core/knowledge/retriever.js`：
  - 构建本地知识 chunks，覆盖结构化规则库、项目 Skill 和关键 `lark-*` Skill。
  - 实现无外部依赖的轻量检索，包含关键词、短语、来源类型和 Skill 路由加权。
  - 支持将检索结果合成为可被现有决策链路消费的知识规则。
- 改造现有匹配链路：
  - `src/app/runDemo.js`：规则未命中时调用 retriever。
  - `src/chatPollMain.js`：轮询触发链路规则未命中时调用 retriever。
  - `src/chatEventMain.js`：事件监听触发链路规则未命中时调用 retriever。
- 增强输出：
  - `src/core/agent/decisionEngine.js` 将召回证据写入 decision。
  - `src/adapters/output/terminalCard.js` 在终端知识卡展示召回证据。
  - `src/adapters/output/larkCardPayload.js` 在飞书卡片展示召回证据。
- 更新 `docs/product-definition.md`，记录 P1 轻量 Knowledge Retriever 第一版实施进展。
- 更新 `docs/next_plan.md`，标记近期任务前三项已完成，并明确下一步继续推进 P2/P4/P5。

### 修改原因

- 当前项目已经具备主动触发和规则卡片推送能力，但知识召回仍主要依赖写死规则，不足以体现 Agent 主动检索内部知识的能力。
- 比赛要求强调基于 API、内部知识库和 OpenClaw/CLI 生态构建主动知识助手，需要让系统在规则未命中时也能自动寻找相关 Skill 与知识来源。
- 先用轻量本地检索验证 Agent 决策链，避免过早引入复杂向量数据库影响 Demo 稳定性。

### 影响范围

- 本地 Demo、轮询触发和事件监听触发都会在规则未命中时启用本地知识召回。
- 规则高置信命中路径保持不变。
- 当前实现只读取本地知识和 Skill 文档，不新增写操作，不改变运行入口。
