# 2026-04-30 项目更改记录（北京时间）

## 00:58

### 修改内容描述

- 修复 `src/chatEventMain.js` 事件订阅退出清理逻辑：
  - 默认使用 `lark-cli event +subscribe --compact --quiet`。
  - 将订阅子进程改为独立进程组，退出时清理整个 `lark-cli` 进程树。
  - 增加 `SIGINT`、`SIGTERM`、`SIGHUP` 退出处理，避免旧监听残留占用单实例订阅锁。
- 为监听式主动触发增加事件漏收补偿：
  - 默认开启轻量历史消息轮询对账。
  - 对事件流未投递但历史消息接口可见的消息进行补处理。
  - 支持 `--no-reconcile-poll` 关闭补偿轮询。
  - 支持 `--reconcile-as`、`--reconcile-interval-ms`、`--reconcile-limit` 调整补偿参数。
- 修复补偿轮询消息缺少 `chat_id` 时被误判为非目标群的问题：
  - 对按群历史接口返回的消息，默认使用 `--source-chat-id` 作为来源群 ID。

### 修改原因

- `lark-cli event +subscribe` 每个应用只允许一个订阅实例，旧监听进程残留会导致新监听无法启动。
- 实测飞书事件长连接在短时间连续消息场景下可能未投递全部消息；历史消息接口可查到完整消息，因此需要使用“事件实时触发 + 轮询最终一致性补偿”的混合方案。
- 历史消息接口返回结构与事件订阅 compact 输出结构不同，需要在代码中做字段兼容。

### 影响范围

- 影响监听式主动触发入口 `npm run demo:chat-event`。
- 不影响轮询式入口 `npm run demo:chat-poll`。
- 默认运行命令不变，但监听入口现在会额外使用 user 身份读取最近群消息做补偿对账；如果 user 身份未授权，会输出 warning，不中断事件订阅。
