### 本次变更

- 更新 `src/core/agent/toolPlanner.js`：
  - 增强 `extractServiceResourceMethod(text)`。
  - 现在除了识别 `service.resource.method` 形式，还能从：

```text
lark-cli <service> <resource> <method>
```

  这种原生命令文本中自动组装出 schema key，例如：

```text
lark-cli im messages send
→ im.messages.send
```

- 更新 `src/core/agent/resultInterpreter.js`：
  - 在已有 `help` 解析基础上，新增 `schema` JSON 结构化解析能力。
  - 解析内容包括：
    - `parameters`
    - 每个参数的 `location`
    - 每个参数是否 `required`
    - `requestBody` 中的 body 字段
  - 新增输出结构：
    - `paramsFields`
    - `bodyFields`
    - 基于 schema 自动生成的 `conclusions`
- 更新 `src/adapters/output/terminalCard.js`：
  - 新增 schema 结果展示：
    - `Schema 参数(--params)`
    - `Schema 体(--data)`
    - `环境实测结论`
- 更新 `examples/lark-cli-error-samples.jsonl`：
  - 追加 1 条 schema 触发样例，用于验证：

```text
lark-cli im messages send --params '{"receive_id":"ou_xxx"}' failed: invalid params: missing requestBody.content
```

  结合 `toolPlanner` 的增强，这条样例现在可以触发：

```text
lark-cli schema im.messages.send
```

  从而佐证 schema 解析改动。
