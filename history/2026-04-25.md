```powershell
node src/main.js \
  --source examples/lark-cli-error-samples.jsonl \
  --knowledge-source lark-docs \
  --lark-doc "https://jcneyh7qlo8i.feishu.cn/docx/YD0bdw3DaoDQTrxomUucfTAOnrd"
```

PowerShell 报错：

```text
一元运算符“--”后面缺少表达式
表达式或语句中包含意外的标记“source”
```

结论：这不是飞书云文档读取失败，也不是 Node 程序参数解析失败，而是 PowerShell 不使用反斜杠 `\` 作为续行符。PowerShell 需要使用反引号 `` ` `` 续行，或者写成单行命令。

### 正确执行方式

推荐单行执行：

```powershell
node src/main.js --source examples/lark-cli-error-samples.jsonl --knowledge-source lark-docs --lark-doc "https://jcneyh7qlo8i.feishu.cn/docx/YD0bdw3DaoDQTrxomUucfTAOnrd"
```

PowerShell 多行写法：

```powershell

  --source examples/lark-cli-error-samples.jsonl `
  --knowledge-source lark-docs `
  --lark-doc "https://jcneyh7qlo8i.feishu.cn/docx/YD0bdw3DaoDQTrxomUucfTAOnrd"
```

### 结论

建议迁移。原因：

- 本项目依赖 `node`、`lark-cli`、shell 脚本和命令行参数，多数文档示例天然更适合 Linux/bash。
- 可以避免 PowerShell 续行符、中文编码、stdout 流式输出交错等 Windows 终端问题。
- Docker 环境更利于复现、提交 Demo、后续部署到云主机或 CI。
- Ubuntu 24.04 适合当前项目，Node 20/22、npm、git、lark-cli 都容易安装。

### 建议环境形态

- 如果只是本机开发：优先用 Docker + bind mount 当前项目目录。
- 如果需要更深度 IDE 集成：可以用 VS Code / Cursor Dev Container。
- 初始阶段不建议立刻复杂化为 docker-compose，除非后续有服务端、Redis、DB 或 webhook listener。
