# 2026-05-04 项目修改记录（北京时间）

## 00:03

### 修改内容描述

- 新增 `src/core/agent/composer.js`，补齐“上下文整理 -> 检索查询 -> 诊断压缩”的 Composer 层：
  - 从事件中提取命令、退出码、stderr/stdout 摘要、cwd 等上下文。
  - 生成统一的检索查询对象。
  - 基于规则命中或 RAG 召回结果，生成更贴近上下文的诊断、建议步骤和下一步命令。
- 将 Composer 接入主链路：
  - `src/app/processKnowledgeEvent.js`
  - `src/app/runDemo.js`
  - `src/chatEventMain.js`
  - `src/chatPollMain.js`
  - `src/core/agent/decisionEngine.js`
- 扩展命令行参数，新增 `--compose-mode` / `--no-compose`，当前默认 `template`。
- 更新 `src/adapters/output/terminalCard.js`，在 Debug View 中展示：
  - 整理模式
  - 检索查询
- 更新 `docs/product-definition.md`，同步记录 Composer 层的加入。

### 修改原因

- 当前项目已经具备规则匹配、RAG 召回、CLI 代理和飞书卡片分发能力，但还缺少一层“把上下文整理成检索问题、再把证据压缩成高密度结论”的 Agent 整合能力。
- 需要先把这层结构补齐，后续再接入真实 LLM 时，才能在不推翻现有链路的前提下自然升级。

### 影响范围

- 终端调试卡片内容更接近“Agent 先整理问题再输出结论”的形态。
- 规则命中与 RAG 路径都会经过 Composer 层，但当前不依赖外部模型或 API key。
- 已验证：
  - `node --check src/core/agent/composer.js`
  - `node --check src/app/processKnowledgeEvent.js`
  - `node --check src/chatEventMain.js`
  - `node --check src/chatPollMain.js`
  - `node --check src/app/runDemo.js`
  - `npm run demo:lark-cli-proxy -- wiki --unknown-flag --agent --push-level all --push-bypass-dedupe`
  - `npm run demo -- --source examples/lark-cli-rag-samples.jsonl --no-quality-report --compose-mode template`

## 02:00

### 修改内容描述

- 新增 `src/core/agent/liveCliEvidence.js`，把当前机器上的 `lark-cli <service> --help` 接成动态证据源：
  - 从事件中提取 `lark-cli` 命令和 service。
  - 只读调用当前环境的 `lark-cli <service> --help`。
  - 将 help 输出解析为 `usage`、`flags` 和结论摘要。
- 更新 Composer 链路：
  - `src/app/processKnowledgeEvent.js` 在构造 composition input 前会收集动态 help 证据。
  - `src/core/agent/composer.js` 将动态 help 融入诊断与建议。
  - `src/core/agent/decisionEngine.js` 将 `liveEvidence` 挂入 decision。
  - `src/adapters/output/terminalCard.js` 在 Debug View 中显示“动态 help”。
- 扩展所有主要入口的命令行参数：
  - `--live-help`
  - `--no-live-help`
  - `--live-help-timeout-ms`
- 修复 `src/core/agent/resultInterpreter.js` 中 `Usage:` 解析过短的问题，现在能正确抓取实际用法，如 `Usage: lark-cli im [command]`。

### 修改原因

- 当前知识源中，官方 skill 已经作为静态证据进入检索，但“当前机器安装的 lark-cli 真实命令形态”还没有进入 Agent 的证据链。
- 需要把静态 skill 与动态 `--help` 结合起来，才能更好地体现“企业知识整合 + 当前环境验证”的 Agent 形态。

### 影响范围

- 终端卡片和代理命令的诊断会同时参考静态 skill 召回与动态 `--help` 结果。
- 不引入写操作，仍然是只读验证。
- 已验证：
  - `node --check src/core/agent/liveCliEvidence.js`
  - `node --check src/core/agent/resultInterpreter.js`
  - `npm run demo:lark-cli-proxy -- wiki --unknown-flag --agent --push-level all --push-bypass-dedupe`
  - `npm run demo:lark-cli-proxy -- im --help --agent --push-level all --push-bypass-dedupe`
## 11:20

### 修改内容描述

- 新增 `src/core/agent/llmComposer.js`，补齐真实 LLM Composer：
  - 通过 OpenAI-compatible `chat/completions` 调用模型。
  - 输入包含上下文、规则命中、RAG 召回结果、工具计划摘要和动态 `lark-cli --help` 证据。
  - 输出归一为 `diagnosis`、`suggestedActions`、`nextCommand`、`routeToSkills`、`confidence`、`uncertainty`。
- 重构 `src/core/agent/composer.js`：
  - `composeKnowledge` 改为异步。
  - 新增 `llm` 模式。
  - 当缺少 API key、请求失败、超时或返回 JSON 不合规时，自动回退到 `template composer`。
  - 回退原因会挂到 `fallbackReason`，供终端 Debug Card 展示。
- 同步更新主链路：
  - `src/app/processKnowledgeEvent.js`
  - `src/app/runDemo.js`
  - `src/main.js`
  - `src/cliWatchMain.js`
  - `src/chatEventMain.js`
  - `src/chatPollMain.js`
- 新增 LLM 参数：
  - `--llm-api-key`
  - `--llm-base-url`
  - `--llm-model`
  - `--llm-timeout-ms`
  - `--llm-temperature`
- 更新 `src/adapters/output/terminalCard.js`，在 Debug View 中显示：
  - `LLM 模型`
  - `回退原因`
- 更新 `README.md` 和 `docs/product-definition.md`，补充真实 LLM Composer 的使用方式与产品定位。

### 修改原因

- 当前项目已经具备规则、RAG 和动态 `--help` 证据链，但最后一层仍是本地模板压缩，不足以完整体现“知识整合与分发 Agent”的 Agent 化整理能力。
- 需要把真实模型接入，但同时保留稳定回退机制，避免演示因为外部密钥或网络问题中断。

### 影响范围

- 所有主入口现在都支持 `--compose-mode llm`。
- 没有 `OPENAI_API_KEY` 时不会报错退出，而是自动回退到 `template_fallback`。
- 已验证：
  - `node --check src/core/agent/llmComposer.js`
  - `node --check src/core/agent/composer.js`
  - `node --check src/app/processKnowledgeEvent.js`
  - `node --check src/app/runDemo.js`
  - `npm run demo:lark-cli-proxy -- wiki --unknown-flag --agent --compose-mode llm --push-level all --push-bypass-dedupe`
  - `npm run demo -- --source examples/lark-cli-rag-samples.jsonl --no-quality-report --compose-mode llm`
