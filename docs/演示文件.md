# 演示文件

这份文件用于比赛现场演示，目标是固定命令、固定问题、固定顺序，避免临场切换思路。

## 1. 三条基础主链路

### 1.1 CLI 主链路

#### 单条命令演示

```bash
npm run demo:competition:cli
```

#### CLI 交互式终端演示

```bash
npm run demo:cli-shell -- \
  --compose-mode llm \
  --push-lark-card \
  --push-chat-id "oc_d32c2e3e2eb66b2efca3ef677620233e" \
  --push-as bot \
  --push-level all \
  --push-bypass-dedupe \
  --retriever-mode hybrid-vector \
  --force-llm-compose \
  --llm-timeout-ms 90000
```

说明：
- 适合演示“开发者在本地终端输入命令，后台自动给出知识建议并推送飞书卡片”。
- 默认建议使用 `release` 视图，必要时再切 `debug`。

### 1.2 云知识 RAG 链路

#### 云知识链路验证

```bash
npm run demo:cloud-calendar
```

#### 云知识 + 飞书卡片推送

```bash
npm run demo:cloud-calendar -- \
  --push-lark-card \
  --push-chat-id "oc_d32c2e3e2eb66b2efca3ef677620233e" \
  --push-as bot \
  --push-level all \
  --push-bypass-dedupe
```

说明：
- 适合演示“本地 skill 之外，还能接飞书云文档 / 云文件夹知识”。

### 1.3 飞书群事件主链路

#### 标准群监听演示

```bash
npm run demo:competition:chat -- \
  --source-chat-id "oc_d32c2e3e2eb66b2efca3ef677620233e" \
  --event-source sdk \
  --push-lark-card \
  --push-chat-id "oc_d32c2e3e2eb66b2efca3ef677620233e" \
  --push-as bot \
  --push-level all \
  --push-bypass-dedupe \
  --force-llm-compose \
  --llm-timeout-ms 90000 \
  --no-reconcile-poll
```

#### 混合检索版群监听演示

```bash
npm run demo:competition:chat -- \
  --source-chat-id "oc_d32c2e3e2eb66b2efca3ef677620233e" \
  --event-source sdk \
  --push-lark-card \
  --push-chat-id "oc_d32c2e3e2eb66b2efca3ef677620233e" \
  --push-as bot \
  --push-level all \
  --push-bypass-dedupe \
  --force-llm-compose \
  --llm-timeout-ms 90000 \
  --retriever-mode hybrid-vector \
  --no-reconcile-poll
```

说明：
- 第一条适合稳定演示主链路。
- 第二条适合强调“混合检索 + LLM 整理”的版本。

## 4. 综合评测命令

### 全量评测

```bash
npm run eval:all -- --show-top3
```

### 混合检索评测

```bash
npm run eval:all -- --show-top3 --retriever-mode hybrid --skip-llm
```

### 向量混合检索质量评测

```bash
npm run eval:quality -- --retriever-mode hybrid-vector --show-top3
```

## 5. 推荐现场演示顺序

建议按下面顺序演示，最顺：

1. `demo:competition:cli`
2. `demo:cli-shell`
3. `demo:cloud-calendar`
4. `demo:competition:chat`
5. `eval:quality -- --retriever-mode hybrid-vector --show-top3`

## 6. 现场注意事项

- 第一次启动 `hybrid-vector` 时，如果需要补 embedding 缓存，等待略长属于正常现象。
- 正式展示时，终端建议使用 `release` 视图，减少调试噪音。
- 如果要排查召回过程，再切到 `debug` 视图。
- 飞书群监听建议优先使用 `sdk` 事件源。
